# Когорты

В блоке страниц «Когорты» располагаются переходы на страницы добавленных когорт и
кнопка <img src={require('/img/rus/add_cohort.png').default} width="20"/> для добавления новой когорты. В когорту можно
объединить пациентов по какому-либо общему признаку (например, по типу патологии или источнику данных).
Когорты подробно описаны в [соответствущем разделе](/cohorts).
Если добавленных когорт нет, то блок содержит окно для группировки пациентов в когорты:

<p align="center">
<img src={require('/img/rus/cohorts_empty.png').default} width="170"/>
</p>

После добавления когорт в блоке появляются переходы на страницы когорт, содержащие только данные о пациентах,
включенных в эти когорты:

<p align="center">
<img src={require('/img/rus/cohorts_added.png').default} width="170"/>
</p>

При наведении курсора на строку когорты на панели навигации становится видимым
элемент <img src={require('/img/rus/cohort_options.png').default} height="20"/>, при нажатии на
который появляются опции "***Изменить***" (открывает окно редактирования когорты) и "***Удалить***" (при нажатии
когорта удаляется). Обратите внимание, что при удалении когорты данные пациентов, входящих в неё, не удаляются.

<p align="center">
<img src={require('/img/rus/cohort_options_list.png').default} width="170"/>
</p>

## Страница когорты {#cohort-page}

Вверху страницы когорты находятся:
- Окно поиска, позволяющее искать пациентов, входящих в когорту, по коду или имени:
<p align="center"><img src={require('/img/rus/search_patient.png').default} width="450"/></p>
- Кнопка <img src={require('/img/rus/upload_samples_btn.png').default} width="150"/>, при нажатии на которую
открывается <a href="/uploading-samples/upload-samples-page">страница множественной загрузки данных</a>.
- Кнопка <img src={require('/img/rus/consolidate_cohort_btn.png').default} width="150"/>,
открывающая форму [создания группового анализа](/cohorts#cohort-analysis) образцов пациентов, входящих в когорту.
- Кнопка <img src={require('/img/rus/add_patient_btn.png').default} width="150"/>,
открывающая окно <a href="/add-patient">создания пациента</a>.

Внизу страницы когорты находятся:
- Информация о количестве пациентов, входящих в когорту: порядок пациентов, отображённых на странице,
(максимум 10 пациентов может
быть показано на странице, так что порядок будет таким: 1-10, 11-20 и т.д.) и суммарное количество пациентов.
- Компонент пагинации: <img src={require('/img/rus/pagination.png').default} width="400"/>, который позволяет
перемещаться между страницами с пациентами когорты. Номер открытой страницы выделен синим кружком. Если нажать на этот
номер, то страница перезагрузится. Чтобы переместиться на страницу с конкретным номером, либо нажмите на этот номер,
либо нажмите на <img src={require('/img/rus/three_points.png').default} width="30"/>, введите нужный номер и нажмите
клавишу ввода или кликните вне области заполнения значения.
Чтобы переместиться на предыдущую по номеру страницу,
нажмите на <img src={require('/img/rus/back.png').default} width="30"/>, а чтобы открыть следующую -
нажмите на <img src={require('/img/rus/next.png').default} width="30"/>.

### Колонки {#cohort-columns}

На странице когорты информация представлена в виде таблицы со следующими колонками:
1. **ПАЦИЕНТ** - информация о пациентах, входящих в когорту, (код, имя, фамилия пациента);
2. **СТАТУС** - статусы наборов образцов каждого
пациента:<br></br><img src={require('/img/rus/in_process_status.png').default} width="22"/> - набор образцов
обрабатывается;<br></br><img src={require('/img/rus/queed_status.png').default} width="14"/> - набор образцов
стоит в очереди
на обработку;<br></br><img src={require('/img/rus/success_status.png').default} width="22"/> - набор образцов
успешно обработан;<br></br><img src={require('/img/rus/problems_status.png').default} width="24"/> - набор образцов
обработан с некритичными проблемами;<br></br><img src={require('/img/rus/failed_status.png').default} width="20"/> - набор
образцов обработан с ошибкой;
3. **ОБРАЗЕЦ** - информация об образцах каждого
пациента:<br></br><img src={require('/img/rus/in_process_sample.png').default} width="18"/> - опухолевый образец
в процессе обработки или в ожидании выполнения необходимых стадий
контрольного образца;<br></br><img src={require('/img/rus/tumor_success.png').default} width="18"/> - успешно
обработанный
опухолевый образец;<br></br><img src={require('/img/rus/tumor_problems.png').default} width="18"/> - опухолевый
образец, обработанный с некритичными
проблемами;<br></br><img src={require('/img/rus/tumor_failed.png').default} width="18"/> - опухолевый
образец, обработанный
с ошибкой;<br></br><img src={require('/img/rus/normal_in_process.png').default} width="18"/> - образец
неопухолевой ткани в
процессе обработки;<br></br><img src={require('/img/rus/normal_success.png').default} width="18"/> - успешно
обработанный образец
неопухолевой ткани;<br></br><img src={require('/img/rus/normal_problems.png').default} width="18"/> - образец
неопухолевой ткани, обработанный с некритичными
проблемами;<br></br><img src={require('/img/rus/normal_failed.png').default} width="18"/> - образец неопухолевой ткани,
обработанный с ошибкой;
4. **КОГОРТА** - название когорты, к которой относится пациент;
5. **СОЗДАН** - дата добавления пациента в формате д.М.гггг;
6. **ИЗМЕНЕН** - дата последних изменений в данных пациента в формате д.М.гггг.

При наведении курсора на строку пациента появляются следующие активные
элементы: <img src={require('/img/rus/upload_in_patient.png').default} width="19"/> позволяет
загрузить данные пациента, а <img src={require('/img/rus/delete_btn.png').default} width="20"/> -
архивировать данные пациента (при архивации пациент когорты перемещается на страницу "*Архив*" из блока "*Пациенты*"
и не отображается на странице когорты).

### Сортировка {#cohort-sorting}

По умолчанию таблица на странице когорты отсортирована от пациентов, добавленных в систему последними,
к пациентам, добавленным первыми,
(т.е. по убыванию значений в колонке "Создан"). Таблицу можно отсортировать по возрастанию или убыванию
значений следующих колонок: "Пациент", "Когорта", "Создан", "Изменен".
Для сортировки таблицы по значениям колонки нажмите на её название (нажмите один раз, чтобы отсортировать
по возрастанию, или два раза, чтобы отсортировать по убыванию). После этого у названия колонки появится
стрелка <img src={require('/img/vv/sort_asc.png').default} width="17"/> (при сортировке по возрастанию)
или <img src={require('/img/vv/sort_desc.png').default} width="17"/> (при сортировке по убыванию).
Для сброса сортировки обновите страницу браузера. Колонки "Пациент" и "Когорта" сортируются по алфавиту,
а колонки "Создан" и "Изменен" - от наиболее ранней к наиболее поздней дате и наоборот.
