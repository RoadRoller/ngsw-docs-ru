# Настройки

Блок "Настройки" включает страницы ["Настройки"](/overview/settings#settings-page),
["Шаблоны отчётов"](/overview/settings#report-templates), ["Панели генов"](/overview/settings#gene-panels),
["Реагенты Capture Kit"](/overview/settings#capture-kits)
и ["Пользовательские аннотации"](/overview/settings#custom-annotations).

### Страница "Настройки" {#settings-page}

Страница содержит предварительные установки настроек для анализа образцов. Предустановки настроек
подробно описаны в [соответствущем разделе](/settings/settings/settings).

<p align="center">
<img src={require('/img/custom-annot/settings_page.png').default} width="900"/>
</p>

:::note На заметку
Количество и вид предустановок настроек зависит от первоначальных настроек, сделанных администратором Системы.
:::

Вверху на странице находится кнопка <img src={require('/img/rus/add_setting.png').default} width="170"/>,
при нажатии на которую открывается окно [создания предустановки настроек](/settings/settings/settings#create-preset).

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ПРЕДВАРИТЕЛЬНАЯ УСТАНОВКА НАСТРОЕК** - имя предустановки настроек;
2. **ОПИСАНИЕ** - описание предустановки настроек;
3. **ПО УМОЛЧАНИЮ** - колонка, в которой
галочкой <img src={require('/img/rus/default_setting.png').default} width="20"/>
отмечена та предустановка настроек, которая выбрана по умолчанию для анализа образцов;
4. **ИЗМЕНЕН** - дата последних изменений предустановки настроек в формате д.М.гггг.

Таблица отсортирована по имени предварительной установки настроек.

При наведении курсора на строку предустановки настроек, не выбранной по умолчанию, появляются следующие активные
элементы: <img src={require('/img/rus/set_as_default.png').default} width="22"/> позволяет установить
предустановку по умолчанию, а <img src={require('/img/rus/delete_btn.png').default} width="20"/> -
удалить предустановку.

### Шаблоны отчётов {#report-templates}

Страница содержит шаблоны отчётов, сформированные для представления результатов
анализа образцов в виде отчётов. Шаблоны отчётов подробно описаны
в [соответствущем разделе](/settings/report-templates/report-templates).

<p align="center">
<img src={require('/img/report_templates/rt_page.png').default} width="900"/>
</p>

:::note На заметку
Количество и вид шаблонов зависит от первоначальных настроек, сделанных администратором Системы.
:::

Вверху на странице находятся
следующие кнопки: <img src={require('/img/rus/template_import.png').default} width="170"/>,
при нажатии на которую открывается
окно <a href="/settings/report-templates/report-templates#import-template">импорта шаблона отчёта</a>,
и <img src={require('/img/rus/add_template.png').default} width="170"/>, открывающая страницу
[создания шаблона](/settings/report-templates/report-templates#add-template).

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ИМЯ** - имя отчёта (дает название вкладке с отчётом на [странице образца](/results/results));
2. **ОПИСАНИЕ** - описание шаблона отчёта;
3. **ПРИМЕНИМ К ОБРАЗЦАМ** - информация о типе образцов, к которым применим шаблон: Любые, Опухолевые,
Неопухолевые;
4. **АКТИВЕН** - колонка, в которой галочкой <img src={require('/img/rus/default_setting.png').default} width="20"/>
отмечены активные отчёты;
5. **ИЗМЕНЕН** - дата последних изменений шаблона отчёта в формате д.М.гггг.

Таблица отсортирована по именам шаблонов отчётов.

При наведении курсора на строку шаблона отчёта появляется
активный элемент <img src={require('/img/rus/delete_btn.png').default} width="20"/>, позволяющий
удалить шаблон.

### Панели генов {#gene-panels}

Страница содержит панели генов, по которым можно проводить фильтрацию выявленных вариантов.
Панели генов подробно описаны в [соответствущем разделе](/settings/gene-panels).

<p align="center">
<img src={require('/img/gene_panels/gene_panels_page.png').default} width="900"/>
</p>

Вверху на странице находятся
следующие кнопки: <img src={require('/img/rus/add_gene_panel.png').default} width="170"/> - открывает окно создания
панели путем выбора генов из списка всех генов или добавления генов, связанных с определенным термином
из <a href="https://hpo.jax.org/app/">Human Phenotype Ontology</a>, <img src={require('/img/rus/import_gene_list.png').default} width="170"/> - открывает
окно создания панели путем импорта
подготовленного списка генов, и <img src={require('/img/rus/import_from_library.png').default} width="170"/> - открывает
окно импорта панели из библиотеки самых распространенных клинических панелей, загруженных администратором системы.

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ПАНЕЛЬ ГЕНОВ** - имя панели генов;
2. **ОПИСАНИЕ** - описание панели генов.

Таблица отсортирована по именам панелей генов.

При наведении курсора на строку панели генов появляется
активный элемент <img src={require('/img/rus/delete_btn.png').default} width="20"/>, позволяющий
удалить панель.

### Реагенты Capture Kit {#capture-kits}

Страница содержит встроенные и загруженные наборы Capture Kit,
которые можно указать при загрузке результатов таргетного секвенирования для более точного и быстрого анализа.
Реагенты capture kit подробно описаны в [соответствущем разделе](/settings/capture-kits).

<p align="center">
<img src={require('/img/capture_kits/ck_page.png').default} width="900"/>
</p>

Вверху на странице находится кнопка <img src={require('/img/rus/upload_ck_btn.png').default} width="170"/>,
при нажатии на которую открывается
окно [загрузки файла с набором capture kit](/settings/capture-kits#upload-capture-kit).

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ИМЯ** - название реагента capture kit (имя загруженного файла capture kit). Нажав на имя
capture kit, можно скачать набор в формате BED;
2. **ТИП** - тип реагента: Встроенный или Пользовательский (загруженный пользователем);
3. **ДЛИНА** - суммарная длина покрытия (п.н.)

Таблица отсортирована сначала по типу реагента (от пользовательских к встроенным реагентам),
а затем по имени реагента capture kit.

При наведении курсора на строку пользовательского набора capture kit появляются следующие активные
элементы: <img src={require('/img/rus/download_kit.png').default} width="20"/> позволяет скачать
загруженный файл capture kit в формате BED, а <img src={require('/img/rus/delete_btn.png').default} width="20"/> -
удалить capture kit.

### Пользовательские аннотации {#custom-annotations}

Страница содержит загруженные пользователем аннотации,
по которым будут аннотироваться найденные варианты. Пользовательские аннотации подробно описаны
в [соответствущем разделе](/settings/custom-annotations).

<p align="center">
<img src={require('/img/rus/custom_annotations_page.png').default} width="900"/>
</p>

Вверху на странице находится кнопка <img src={require('/img/rus/upload_custom_annot.png').default} width="170"/>,
при нажатии на которую открывается
страница [добавления пользовательской аннотации](/settings/custom-annotations#add-custom-annotation).

Информация на странице представлена в виде таблицы со следующими колонками:
1. **ИМЯ** - имя пользовательской аннотации (может быть задано или изменено до момента,
когда аннотация успешно обработана, либо совпадает с имененем загруженного файла аннотации);
2. **СТАТУС** - статусы загруженных
аннотаций:<br></br><img src={require('/img/rus/process_not_started.png').default} width="160"/> - процесс
обработки аннотации ещё не
был <a href="/settings/custom-annotations#perform-source-preparation">запущен</a>;<br></br><img src={require('/img/rus/In_progress.png').default} width="140"/> - процесс
обработки аннотации запущен, но ещё не
завершён;<br></br><img src={require('/img/rus/success_annot.png').default} width="100"/> - процесс обработки аннотации
успешно завершён;<br></br><img src={require('/img/rus/problems_annot.png').default} width="100"/> - процесс
обработки аннотации успешно завершён, но во время обработки возникли некритичные проблемы (детали можно посмотреть
на странице пользовательской
аннотации в <a href="/settings/custom-annotations#perform-source-preparation">разделе "Статус"</a>);<br></br><img src={require('/img/rus/failed_annot.png').default} width="85"/> - процесс
обработки аннотации завершился неудачно (детали можно посмотреть на странице пользовательской
аннотации в <a href="/settings/custom-annotations#perform-source-preparation">разделе "Статус"</a>);
3. **ТИП** - тип аннотации:<br></br><img src={require('/img/rus/cosmic.png').default} width="110"/> - аннотация
с таким типом используется в качестве базы
COSMIC;<br></br><img src={require('/img/rus/pathogenicity.png').default} width="130"/> - аннотация с таким
типом используются в качестве базы патогенности (данные о классах патогенности
вариантов);<br></br><img src={require('/img/custom-annot/freq_base.png').default} width="140"/> - аннотация с таким
типом используется в качестве базы частот (данные о встречаемости вариантов в
популяции);<br></br><img src={require('/img/custom-annot/ruseq_button.png').default} width="140"/>- аннотация с таким
типом используется в качестве базы RUseq;
4. **ИЗМЕНЕН** - дата последних изменений пользовательской аннотации в формате д.М.гггг.

Таблица отсортирована по дате добавления пользовательских аннотаций (от аннотаций, добавленных последними,
к аннотациям, добавленным первыми).

При наведении курсора на строку пользовательского аннотации появляется
активный элемент <img src={require('/img/rus/delete_btn.png').default} width="20"/>, который позволяет
удалить аннотацию.
