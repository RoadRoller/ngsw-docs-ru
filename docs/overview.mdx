# Знакомство с сервисом

import Login from '/docs/imported_components/_login.mdx';

<Login />

После входа в систему вы попадете на главную страницу сервиса “**Все пациенты**”.
Если вы не пользовались сервисом раньше, страница будет пустой:

<p align="center">
<img src={require('/img/rus/no_patients_page.png').default} width="1000"/>
</p>

Впоследствии вы увидите здесь всех пациентов, добавленных в этот аккаунт на данный момент:

<p align="center">
<img src={require('/img/version_1.02/1-all-patients-page.png').default} width="1000"/>
</p>

Слева на странице, на боковой панели навигации находятся следующие разделы сервиса (для перехода в необходимый раздел
необходимо нажать левой кнопкой мыши на соответствущее название страницы):

1. Блок "***Пациенты***", который включает страницы:
- "***Все пациенты***" - содержит всех добавленных в систему пациентов и их данные.
- “***Обрабатывается***” - содержит тех пациентов, данные которых находятся в процессе обработки.
- “***Проблемы***” - содержит тех пациентов, при обработке данных которых возникли проблемы, или пациенты с
особенностями данных.
- “***Архив***” - содержит тех пациентов, данные которых были архивированы. Вы можете перенести сюда
неактуальные данные или данные, которые вы хотите удалить. Пациенты, находящиеся в "Архиве", не отображаются
на странице "Все пациенты".

<summary>Нажмите, чтобы узнать подробнее о виде, в котором представлена информация на страницах

пациента: <img src={require('/img/rus/in_process_status.png').default} width="20"/> - набор образцов
обрабатывается; <img src={require('/img/rus/success_status.png').default} width="25"/> - набор образцов
успешно обработан; <img src={require('/img/rus/problems_status.png').default} width="28"/> - набор образцов
обработан с некритичными проблемами; <img src={require('/img/rus/failed_status.png').default} width="25"/> - набор
образцов обработан с ошибкой;
- «ОБРАЗЕЦ» - информация об образцах каждого
пациента: <img src={require('/img/rus/in_process_sample.png').default} width="22"/> - опухолевый образец

архивировать данные пациента.

элементы: <img src={require('/img/rus/restore_btn.png').default} width="22"/> позволяет восстановить данные пациента,
а <img src={require('/img/rus/delete_btn.png').default} width="18"/> - удалить их.

Вверху страниц блока находится окно поиска, позволяющее искать пациентов по коду или имени:
<p align="center"><img src={require('/img/rus/search_patient.png').default} width="400"/></p>

кнопки: <img src={require('/img/rus/upload_samples_btn.png').default} width="150"/>, при нажатии на которую открывается
страница множественной загрузки данных, и <img src={require('/img/rus/add_patient_btn.png').default} width="150"/>,
открывающая окно <a href="/add-patient">создания пациента</a>.

На странице «<i>Архив</i>» вверху находится
кнопка <img src={require('/img/rus/clear_archive_btn.png').default} width="150"/>, при нажатии на которую удаляются
все архивированные данные пациентов.
</details>
