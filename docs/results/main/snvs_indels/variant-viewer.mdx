# Variant Viewer

## Знакомство с Variant Viewer

Variant Viewer - это встроенный модуль для просмотра и анализа вариантов. Доступен на главной странице образца
(вкладка **[Главное](/results/main/main)**).

На странице образца ***опухолевой*** ткани вы найдете два Variant Viewer: с ***соматическими*** и с
***герминальными*** мутациями, выявленными у этого образца:

<p align="center">
<img src={require('/img/vv/germline_and_somatic_vv.png').default} width="600"/>
</p>

На странице образца ***нормальной*** ткани представлен Variant Viewer только с ***герминальными*** мутациями:

<p align="center">
<img src={require('/img/vv/germline_only.png').default} width="600"/>
</p>

## Основная страница

Нажав на кнопку <img src={require('/img/results/vv_btn.png').default} width="130"/>,
вы попадете на основную страницу модуля:

<p align="center">
<img src={require('/img/vv/main_table.png').default} width="950"/>
</p>

:::info На заметку
Вид некоторых функций Variant Viewer
может быть изменён на [странице "Настройки пользователя"](/settings/profile#variant-viewer).
:::

Основным объектом Variant Viewer является таблица с вариантами - каждый вариант на новой строчке:

<p align="center">
<img src={require('/img/vv/snv_row.png').default} width="950"/>
</p>

### Информация о варианте

Помимо представленной в таблице информации (см.
[Описание колонок таблицы](/results/main/snvs_indels/variant-viewer#описание-колонок-таблицы)), для каждого варианта
есть несколько встроенных модулей с дополнительной информацией:

1. **Панель детальной информации** о варианте. Чтобы увидеть панель, нажмите на строку варианта:

<p align="center">
<img src={require('/img/vv/details_panel.png').default} width="950"/>
</p>

Подробнее о панели и её вкладках можно почитать
в [соответствующем разделе](/results/main/snvs_indels/snv_details_panel).

2. **Страница детальной информации** о варианте - здесь можно добавить интерпретацию для варианта, найти
дополнительные сведения о варианте и многое другое. Для открытия страницы
нажмите на <img src={require('/img/vv/details_info_btn.png').default} width="25"/> в строке варианта.
Подробнее о странице детальной информации можно почитать
в [соответствующем разделе](/results/main/snvs_indels/snv_detaile_page).

3. **ACMG классификация** варианта - страница, на которой представлены критерии для интерпретации
вариантов, разработанные [ACMG](https://www.acmg.net/). Чтобы попасть на страницу,
нажмите на кнопку патогенности (<img src={require('/img/vv/acmg_var_btn.png').default} width="42"/>, если класс
патогенности этого варианта ранее не определялся)
в строке варианта (колонка "**Патогенность**") и в появившемся диалоге нажмите на "**ACMG Классификация**":

<p align="center">
<img src={require('/img/vv/acmg_on_dialog.png').default} width="300"/>
</p>

Подробнее о странице "ACMG классификация" можно почитать
в [соответствующем разделе](/results/main/snvs_indels/pathogenicity#acmg-классификация-варианта).

4. **Модуль визуализации варианта в геноме (IGV)**. Чтобы открыть встроенный модуль,
нажмите на <img src={require('/img/vv/igv_var.png').default} width="33"/> в строке варианта.

### Закрепление вариантов

Нужные варианты можно закрепить. Для этого
нажмите на <img src={require('/img/vv/pin_variant.png').default} width="28"/> в строке варианта.
Закрепленный вариант окажется сверху в таблице (строка подсвечена зелёным):

<p align="center">
<img src={require('/img/vv/pinned_variant.png').default} width="950"/>
</p>

Чтобы открепить вариант, нажмите на ту же кнопку <img src={require('/img/vv/unpin.png').default} width="28"/>
в строке варианта.

:::note На заметку
Закрепление варианта сохраняется при выходе со страницы Variant Viewer.
:::

### Включение вариантов в отчёт

Включенные в отчёт варианты будут входить в блоки отчётов с типом
["Выбранные пользователем SNVs/Indels для добавления в отчет"](/settings/report-templates/report-templates#типы-блоков-шаблона-отчёта-и-их-редактирование).
Для включения варианта в отчёт нажмите на <img src={require('/img/vv/include_to_report.png').default} width="25"/> в
строке варианта. Для исключения варианта из отчёта нажмите
на ту же кнопку <img src={require('/img/vv/remove_from_report.png').default} width="25"/> в строке варианта.
Также включенные в отчёт варианты можно удалить из отчёта на странице самого отчёта.

:::info На заметку
При [определении](/results/main/snvs_indels/pathogenicity) варианта как патогенного или вероятно патогенного
он автоматически добавляется в отчёт.
:::

### Сортировка и поиск вариантов

Таблицу можно ***сортировать*** по возрастанию или убыванию значений следующих колонок: "Локус", "Тип мутации",
"Позиция", "Экзон", "Глубина (Альт/Реф)", "Эффект варианта", "gnomAD AF", "Патогенность". Для этого нажмите на
название соответствующей колонки (один раз, чтобы отсортировать по возрастанию, два раза - по убыванию).

***Поиск вариантов*** в Variant Viewer описан в соответствующих разделах
[Базовый поиск вариантов](/results/main/snvs_indels/basic_filtering) и
[Расширенный поиск вариантов](/results/main/snvs_indels/advanced_filtering).

<p align="center">

и выберите опцию "Только закрепленные в CSV":

## Описание колонок таблицы

По умолчанию таблица включает следующие колонки (набор и порядок колонок можно изменить - то, как это
сделать, описано [здесь](/results/main/snvs_indels/edit_table_columns)):

1. **Локус** - название гена, в котором располагается вариант.
2. **Тип мутации** (соматическая или герминальная).

:::note На заметку
Колонка "Тип мутации" отсутствует в [Variant Viewer с герминальными мутациями](/results/main/snvs_indels/variant-viewer)
и в Variant Viewer с соматическими мутациями, если включен фильтр "Origin является SOMATIC".
:::

3. **Позиция** - координаты варианта в геноме (хромосома + начальная позиция).
4. **HGVSp** - аминокислотная замена в [номенклатуре HGVS](https://varnomen.hgvs.org/) (префикс “p.” (protein) +
референсная аминокислота + позиция аминокислоты в белке + новая аминокислота, получившаяся в результате замены).
При наведении курсора на значение колонки "HGVSp" для варианта можно увидеть транскрипт, для которого была
предсказана замена, и HGVSp с трехбуквенным обозначением аминокислот:

<p align="center">
<img src={require('/img/vv/hgvsp_tooltip.png').default} width="350"/>
</p>

4. **Замена** - нуклеотидная замена (референсный аллель > альтернативные аллели/аллель).
5. **HGVSc** - нуклеотидная замена в [номенклатуре HGVS](https://varnomen.hgvs.org/) (геномная позиция замещенного
нуклеотида + референсный аллель > альтернативный аллель). Начинается с префикса “c.” (coding; для замены в
кодирующей последовательности) или “n.” (non-coding; для замены в некодирующей последовательности). При наведении
курсора на значение колонки "HGVSс" для варианта можно увидеть, для какого транскрипта предсказана замена (приведены
идентификаторы транскриптов из Ensembl (ENST...) и RefSeq (NM_)):

<p align="center">
<img src={require('/img/vv/hgvsc_tooltip.png').default} width="300"/>
</p>

6. **Экзон** - номер экзона в транскрипте, который вариант затронул.
7. **Глубина (Альт/Реф)** - глубина секвенирования; общее количество прочтений последовательности, перекрывающих
позицию варианта, для этого образца. **Альт** - количество раз, когда считывался альтернативный нуклеотид,
**Реф** - количество раз, когда считывался референсный нуклеотид. Также приведен процент доли количества прочтений
альтернативного аллеля среди всех прочтений (Альт/Глубина).

Ячейки колонки раскрашены в зависимости от значения глубины:
- зелёным, если глубина > 60;
- жёлтым, если глубина ≤ 60;
- красным, если глубина < 20.

8. **GT** (генотип) - значения аллелей (0 - референсный аллель, 1 - первый альтернативный аллель, 2 - второй
альтернативный аллель и т.д.), разделенные ”/” (для нефазированного генотипа) или “|” (для фазированного генотипа).
9. **Эффект варианта** - последствия варианта для генов.

Ячейки колонки раскрашены в зависимости от значения частоты:
- зелёным, если AF ≥ 0.015;
- жёлтым, если 0.01 ≤ AF < 0.015;
- красным, если AF < 0.01.
