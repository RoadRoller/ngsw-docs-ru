# Отчет на основе данных ClinVar с рекомендациями

Отчет включает ***патологии***, риск развития которых у пациента повышается в связи с наличием
в образце пациента определенных патогенных вариантов, и ***рекомендации***,
которые могут помочь пациенту снизить риски возникновения этих патологий.

Чтобы получить такой отчёт для образца,
[создайте шаблон отчета](/settings/report-templates/report-templates#добавление-шаблона-отчета), включающий
блок "SNVs/Indels по фенотипам ClinVar".

Отчёт разбит на три основных раздела:

### 1. Список патологий и общая рекомендация

- Список патологий (фенотипов ClinVar), риск развития которых у пациента повышается в связи с наличием
в образце пациента определенных патогенных вариантов;
- Общая рекомендация для пациента (консультация врача генетика).

### 2. Подробный отчет

Подробный отчет о патологиях, перечисленных в первом разделе. Для каждой патологии указано следующее:
- **Название патологии** (соответствует полю "Имя в отчете"
из [шаблона](/settings/report-templates/report-templates#типы-блоков-шаблона-отчета-и-их-редактирование)).
- **Варианты**, выявленные в образце, которые связаны с данной патологией (с указанными в шаблоне MedGen ID
и/или именем в ClinVar). Кроме этого, **каждый вариант соответствует следующим условиям**:
1. Патогенный или вероятно патогенный по данным ClinVar (поле "Clinical significance");
2. Не гомозиготный по референсу (содержит хотя бы один альтернативный аллель);
3. Суммарная частота аллеля в gnomAD выше порога частоты крайне редких патогенных вариантов и ниже
порога частоты слишком частых патогенных вариантов (пороги задаются
в [шаблоне](/settings/report-templates/report-templates#типы-блоков-шаблона-отчета-и-их-редактирование)).
- **Рекомендации**, которые могут помочь снизить риски возникновения данной патологии (добавляются
в [шаблоне](/settings/report-templates/report-templates#типы-блоков-шаблона-отчета-и-их-редактирование)).

### 2. Исключенные варианты

Представляет варианты, которые были **исключены из подробного отчета** о патологиях, по одному из следующих критериев
(при этом всем перечисленным выше условиям для попадания в отчёт варианты удовлетворяют):
1. **Критерий 1**: варианты, которые имеют **слишком высокую частоту встречаемости** (более 15%) среди
представителей человеческой популяции (противоречит данным о возможной патогенности этих вариантов);
2. **Критерий 2**: варианты, которые имеют **генотип 1/1** и
**частоту встречаемости среди представителей человеческой популяции менее 1%**.
Согласно [публикации](https://www.bmj.com/content/372/bmj.n214), такие варианты
рекомендуется исключать, поскольку они могли быть выявлены в результате ошибки генотипирования, проведенного
с помощью технологии микрочипов.
3. **Критерий 3**: варианты, которые имеют **достоверность выявления ниже установленного порога достоверности**
(порог задается в [шаблоне](/settings/report-templates/report-templates#типы-блоков-шаблона-отчета-и-их-редактирование)).<br></br>Достоверность
рассчитывается как *(1 - Bin(n; m; p; 1))*,<br></br>где
*Bin(n; m; p; 1)* - это рассчитанная на основе биномиального распределения вероятность того, что вариант
встретится ровно *n* раз среди *m* представителей (образцов пациентов) популяции;<br></br>
*n* - количество представителей популяции (образцов пациентов), у которых был выявлен данный вариант;<br></br>
*m* - общее количество представителей (образцов пациентов) в популяции;<br></br>
*p* - вероятность встретить вариант среди представителей человеческой популяции (суммарная частота аллеля в базе
данных [gnomAD](https://gnomad.broadinstitute.org/)). Если данных о частоте в популяции нет, то вероятность
принимается за 0,001.<br></br>
Таким образом, из отчета исключаются варианты, которые встретились более, чем *n* раз, среди *m* образцов популяции,
но при этом имеют низкую частоту в человеческой популяции, т.е. информация о возможной патогенности таких
вариантов противоречит слишком частой встречаемости среди представителей (образцов пациентов) популяции.<br></br>
Значения *n* и *m* берутся
из [пользовательской базы частот](/settings/custom-annotations#2-определение-типа-пользовательской-аннотации)
(если она не была загружена в систему, фильтрация вариантов по критерию 3 происходить не будет).

#### Каждый вариант в отчете представлен в следующем виде:

<p align="center">
<img src={require('/img/results/variant.png').default} width="900"/>
</p>

1. **Ген** - название гена, в котором располагается вариант.
2. **Идентификатор варианта** - идентификатор варианта из базы данных [dbSNP](https://www.ncbi.nlm.nih.gov/snp/).
3. **Референсный аллель** варианта.
4. **Альтернативный аллель** варианта.
5. **gnomAD AF** - суммарная частота аллеля в базе данных [gnomAD](https://gnomad.broadinstitute.org/).
6. **Хромосома** - хромосома, на которой находится вариант.
7. **Начало** - начальная позиция варианта на хромосоме.
8. **Конец** - конечная позиция варианта на хромосоме.
9. **Аминокислотная замена** - номер экзона в транскрипте, который был затронут вариантом; аминокислотная и
нуклеотидная замены в [номенклатуре HGVS](https://varnomen.hgvs.org/). Аминокислотная: префикс “p.” (protein) +
референсная аминокислота + позиция аминокислоты в белке + новая аминокислота, получившаяся в результате замены.
Нуклеотидная: префикс “c.” (coding; для замены в кодирующей последовательности) или “n.” (non-coding; для замены
в некодирующей последовательности) + геномная позиция замещенного нуклеотида + референсный аллель > альтернативный
аллель.
10. Указание, в какой форме была обнаружена мутация: **гетерозиготной** или **гомозиготной**.
11. Для **исключенных** из подробного отчета вариантов - указание критерия, по которому они были исключены.
