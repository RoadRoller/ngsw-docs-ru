# Запуски

Раздел "Запуски" дает возможность загрузить несколько образцов без определения для каждого образца пациента,
к которому он относится. Ещё одним отличием Запуска от
[Загрузки образцов в пациента](uploading-samples/normal-sample/normal-sample) является построение общего
[Variant Viewer](/runs#variant-viewer-запуска), который содержит варианты, обнаруженные во всех образцах Запуска.

Чтобы открыть страницу с запусками, перейдите на страницу "**Запуски**" из одноименного блока страниц
через панель навигации слева:

<p align="center">
<img src={require('/img/runs/run_page.png').default} width="900"/>
</p>

### Создание запуска

Для перехода к форме создания запуска нажмите на <img src={require('/img/runs/add_run.png').default} width="150"/>.
В появившемся окне введите название запуска:

<p align="center">
<img src={require('/img/runs/add_run_modal.png').default} height="350"/>
</p>

и выберите тип образцов запуска (***NORMAL*** - если запуск будет состоять преимущественно из образцов здоровой ткани,
или ***TUMOR*** - если из образцов опухолевой ткани):

import NormalTissueInfo from '/docs/imported_components/_normal_tissue_info.mdx';

<SectionDef />

<p align="center">
<img src={require('/img/runs/sample_type.png').default} height="350"/>
</p>

:::info На заметку
Тип любого образца в запуске можно будет поменять после загрузки.
:::

Для создания запуска нажмите <img src={require('/img/runs/create_btn.png').default} width="130"/>.
Вы окажетесь на странице загрузки образцов в запуск:

<p align="center">
<img src={require('/img/runs/new_run_page.png').default} width="900"/>

### Настройки анализа

Выберите нужную настройку анализа образцов.
Обратите внимание, что настройка анализа будет применена ко всем образцам запуска и должна соответствовать
выбранному типу образцов запуска (TUMOR или NORMAL).
Например, настройку [Germline variant analysis](/uploading-samples/normal-sample/germline_preset) нельзя
использовать, если был выбран тип образцов "TUMOR", поскольку она подходит только для анализа образцов здоровой ткани.

#### Смена типа образца

Тип каждого образца указан справа от карточки образца и соответствует типу, который был указан при создании
запуска. Выбранный тип выделен синей рамкой.
При необходимости тип любого образца можно поменять, нажав на нужный тип левой кнопкой мыши:

<p align="center">
<img src={require('/img/runs/change_sample_type.png').default} width="500"/>
</p>

Тип любого образца можно будет поменять и после старта анализа.

### Запуск анализа

После добавления всех образцов нажмите <img src={require('/img/runs/analyze_samples_btn.png').default} width="200"/>,
чтобы начать анализ образцов запуска.

### Страница запуска

Сразу после старта анализа вы попадете на страницу запуска:

<p align="center">
<img src={require('/img/runs/beginning.png').default} width="900"/>
</p>

Сверху указаны название запуска и его описание, которые можно поменять нажатием на них и вводом нового значения.
Ниже представлена таблица, в которой каждая строка - это образец запуска.

### Образцы запуска

#### 1. Результаты анализа образца

Результаты для каждого отдельного образца запуска находятся на странице образца. Страница образца запуска не отличается
от страницы образца пациента. Описание результатов можно посмотреть [здесь](/results/results).

#### 2. Изменение хода анализа образца

Можно менять настройки анализа (вкладка "**Параметры**") и перезапускать процессинг образцов. В данном случае изменения
в настройках влияют на ход анализа только конкретного образца, для которого их меняют.

#### 3. Смена типа образца

Для любого образца из запуска можно поменять тип образца (TUMOR или NORMAL).
Для этого нужно нажать на значение колонки ТИП ОБРАЗЦА в строке образца:

<p align="center">
<img src={require('/img/runs/sample_type_value.png').default} width="1000"/>
</p>

и поменять тип на нужный:

<p align="center">
<img src={require('/img/runs/change_type.png').default} width="900"/>
</p>

#### 4. Определение пациента для образца

Для любого образца из запуска можно определить пациента, у которого был взят образец.
Для этого нажмите на "-" в колонке пациента в строке образца:

<p align="center">
<img src={require('/img/runs/patient_field.png').default} width="900"/>
</p>

и в появившемся списке выберите нужного пациента или добавьте нового:

<p align="center">
<img src={require('/img/runs/patient_options.png').default} width="900"/>
</p>

После этого данные об образце будут представлены не только ***на странице запуска***:

<p align="center">
<img src={require('/img/runs/patient_1.png').default} width="900"/>
</p>

но и ***на странице этого пациента***:

<p align="center">
<img src={require('/img/runs/patient_sample.png').default} width="900"/>
</p>

При этом страница образца одна и та же - независимо от того, зашли вы на неё со страницы Запуска или со страницы
Пациента. Любые изменения, включая удаление образца, отобразятся на обеих страницах.

#### 5. Удаление образца

Для удаления образца нужно навести курсор на его строку и нажать на появившийся значок корзины:

<p align="center">
<img src={require('/img/runs/delete_sample.png').default} width="900"/>
</p>

### Variant Viewer запуска

В запуске есть возможность посмотреть варианты, обнаруженные во всех образцах запуска, для которых проводилась и успешно
закончилась стадия коллинга вариантов (Выявление соматических или герминальных SNVs/Indels).
Для открытия Variant Viewer запуска нажмите
на <img src={require('/img/runs/vv_btn.png').default} width="120"/> на странице запуска:

<p align="center">
<img src={require('/img/runs/vv_of_run.png').default} width="900"/>
</p>

Variant Viewer запуска - это стандартный для Genomenal Variant Viewer (подробное описание этого сервиса можно
найти [здесь](/results/main/snvs_indels/variant-viewer)) с дополнительной колонкой "ОБРАЗЕЦ" -
колонкой с названиями образцов, в которых был найден вариант.
Нажав на название образца, вы попадете на страницу Variant Viewer этого образца.

<p align="center">
<img src={require('/img/runs/sample_vv.png').default} width="900"/>
</p>

Обратите внимание, что варианты Variant Viewer запуска - результат объединения вариантов, полученных
для всех образцов запуска, а не их пересечения. Одинаковые по всем параметрам варианты из разных образцов
всегда будут отличаться по значению колонки "ОБРАЗЕЦ".

По колонке "ОБРАЗЕЦ" можно фильтровать варианты. Для этого нажмите на соответствующее окно фильтрации:

<p align="center">
<img src={require('/img/runs/sample_filtering.png').default} width="900"/>
</p>

и выберите название нужного образца:

<p align="center">
<img src={require('/img/runs/sample_choice.png').default} width="300"/>
</p>

Variant Viewer запуска и образцов запуска связаны между собой: если вы закрепите или проинтерпретируете вариант или
определите его патогенность по критериям ACMG в Variant Viewer образца запуска, результат тут же появится в Variant
Viewer запуска, и наоборот.

### Архивация, восстановление и удаление данных запуска

Для ***архивации*** данных запуска наведите курсор на строку запуска и нажмите на появившийся значок корзины:

<p align="center">
<img src={require('/img/runs/archive_run.png').default} width="900"/>
</p>

Запуск будет перемещен на страницу "**Архив**" из блока страниц "Запуски". Перейдя на эту страницу, вы можете:

1. ***Восстановить*** данные запуска, наведя курсор на строку запуска и нажав на выделенный значок:

<p align="center">
<img src={require('/img/runs/restore_run.png').default} width="900"/>
</p>

Тогда запуск будет перемещен обратно на страницу "Запуски" из одноименного блока страниц.

2. ***Удалить*** данные запуска, наведя курсор на строку запуска и нажав на появившийся значок корзины:

<p align="center">
<img src={require('/img/runs/delete_run.png').default} width="900"/>
</p>

Кроме того, вы можете удалить все архивированные данные запусков,
нажав на <img src={require('/img/runs/clear_archive_btn.png').default} width="150"/>.
